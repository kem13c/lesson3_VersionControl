Highlight #1 What is Version Control

Version control can be described as a system where you can store files simultaneously as you edit or make changes to a file. Also version control allows one to go back to a specific file or alteration thats been made and pushed to the source in the past. This sytem enables the you to revert a project of file to one made and saved and pushed to the system prior to the most recent push. It is the management of changes to documents, computer programs, large web sites, and other collections of information.

Highlight #2 Basic Git Commands

We have learned that git is a storage of snaps from a peice of work throughout the projects changes. The most basic GIT commands consist of the following: git commit; git branch; git checkout. git commit- records changes to the repository. git branch- lists creates or deletes branches. git checkout- switch branches or restore working tree files.

Highlight #3 Remote Repository

A remote repository is a verion of the project or work that is stored elsewhere and uses Git commands. These locations of the many versions of a project are stored in locations such as on the internet. Remote repositories can be deleted or removed if they are no longer considered valid. You can also add a repository if needed. Repositories manage multiple branches and move their locations.

Highlight #4 Committing Files

To commit new files on git hub you must create a new repository. Then you must utilize both the terminal in cloud 9 and type commands to push the files to the connected repository. Each command begins with a $ git. Then commands such as $git init and $git commit are used.
